##Docker

  • Docker image 镜像

Docker 镜像是 Docker 容器运行时的只读模板，每一个镜像由一系列的层 (layers) 组成。Docker 使用 UnionFS 来将这些层联合到单独的镜像中。UnionFS 允许独立文件系统中的文件和文件夹(称之为分支)被透明覆盖，形成一个单独连贯的文件系统。正因为有了这些层的存在，Docker 是如此的轻量。当你改变了一个 Docker 镜像，比如升级到某个程序到新的版本，一个新的层会被创建。因此，不用替换整个原先的镜像或者重新建立(在使用虚拟机的时候你可能会这么做)，只是一个新 的层被添加或升级了。现在你不用重新发布整个镜像，只需要升级，层使得分发 Docker 镜像变得简单和快速。

  • Docker repository仓库

Docker 仓库用来保存镜像，可以理解为代码控制中的代码仓库。同样的，Docker 仓库也有公有和私有的概念。公有的 Docker 仓库名字是 Docker Hub。Docker Hub 提供了庞大的镜像集合供使用。这些镜像可以是自己创建，或者在别人的镜像基础上创建。Docker 仓库是 Docker 的分发部分。
注：registry是注册服务器（库），负责管理仓库，而repository是仓库

  • Docker container容器

Docker 容器和文件夹很类似，一个Docker容器包含了所有的某个应用运行所需要的环境。每一个 Docker 容器都是从 Docker 镜像创建的。Docker 容器可以运行、开始、停止、移动和删除。每一个 Docker 容器都是独立和安全的应用平台，Docker 容器是 Docker 的运行部分。

  • 因为namespace 从进程、网络、IPC、文件系统、UTS 和用户角度的隔离，一个 container 就可以对外展现出一个独立计算机的能力

It’s wrong that ConvNets need a lot of data to train.
事实上少量的数据同样可以，一般来说很少完全地开始进行卷积神经网络的训练，通常会先进行预训练和细调。

###预训练
可以先在一些大的数据集训练如ImageNet data，再去在我们所感兴趣的数据中训练。
首先得到ImageNet进行预训练的网络，再将最上方的层（分类器）移除，然后将这个卷积神经网络看成是一个固定特征提取器来进行训练，把这个特征提取器置于你的数据集上方，然后将替换原先作为分类器的层。

再根据你的数据集大小来确定如何对卷积网络的最后一层进行训练或者对整个网络的一部分反向传播进行微调。如果有更多的数据，则可以在整个网络进行更深的反向传播。

##第一个分类器：Nearest Neighbor Classifier
CIFAR-10用来训练测试NNC

speed depend on data size
近邻算法是一个即时的训练方法，但同时是一个昂贵的测试方法，因为需要遍历。但是在训练的时候经过大量的计算，事实上对于每个测试图片都进行恒量的计算而无视data szie。

有很多种距离计算的算法如曼哈顿距离（比较绝对值）
如何进行距离的计算：
distance：一个用来进行控制的离散选择，这个距离选择叫超参数hyperparameter
k：另一种超参数，在近邻算法分类器背景下，k-nearest neighbor classifier的k
How to set the hyperparameter？->Literally, It’s very problem-dependent. Must try them all out and see what works best。

测试集数算法泛化程度的代表，事实上，不应只相信测试集，而是忘记测试数据，这样才能得知算法对于其他数据的泛化程度（对其他数据的有效性）。 有时候在训练过程中表现很好而测试过程中不那么好，这就是泛化能力不够，说明这个分类器已经过拟合。

将训练集分成“fold”，比如说把训练数据分成五等份，也就是说使用20%训练数据来作为想象中的验证集来调整超参数。从而对不同超参数的学习效率进行对比，通过比对不同K值下取多少个近邻值才能得出准确结果。

###Cross-validation
如果对数据量不满意，可以用cross-validation，循环选择测试验证折

利用交叉验证法找出最佳k值

18.8.26About Torch

Posted on 2018-08-26

静态的计算模块如Tenorflow，大部分时候，是预先搭建好的一个计算系统，后期无法改动（除了例外dynamic_rnn()，总体上还是一个静态思维）。当然很多情况下这样足矣。

不动结构当然可以提高效率，但是一旦计算流程不是静态的，计算图要变动，最典型就是RNN：有时候RNN的time stepp不会一样，或者在training和testing 时候，batch_size和time_step也不一样。

#Overfitting
一般在回归分类里，在样本不足的情况下，有时候误差相对较大的线反而能正确反映趋势，而误差较小的线过于拟合坐标点而导致弯弯曲曲的。

解决办法：

  1. 增大数据量，大部分过拟合产生原因就是数据量不足
  2. 运用正规化regularization，通过在计算误差时做手脚，来使得W的变化会受到cost的惩罚
  3. Dropout regularization，在训练时，随机忽略掉一些神经元和神经联结，使得神经网络在不完整的情况下训练一遍，在下一次再随机忽略掉另一些。这样可以使的预测结果不会依赖于其中某些特定的神经元。

#Batch Normalization
在神经网络中，数据分布对训练会产生影响。如某个神经元x的值为1，某个Weights初始值为0.1，这样一层神经元计算后Wx=0.1，如果x=20，则Wx=2。接下来当加上一层激励函数（引入非线性因素，解决线性模型无法解决的问题）的时候，激活这个Wx值的时候，如果使用的是tanh的激励函数，Wx的激活值变成～0.1和～1，接近1的部分已经处在了激励函数的饱和阶段，就是说无论x再怎么扩大，tanh输出值也是接近1。

normalization预处理数据使得x变化范围不会太大，让输入值经过激励函数的有效区间，但是上面所说的不敏感问题不仅仅发生在输入层，在隐藏层也经常发生。批处理batch normalization正是用于处理隐藏层的输入结果。

#生成网络
生成网络利用随机数来生成结果
GAN里有两个部件：Generator&Discriminator
Generator会根据随机数来生成有意义的数据，Discriminator会学习如何判断哪些是真实数据，哪些是生成数据，再将学习经验反向传递给Generator，让Generator能根据随机数生成更像真实数据的数据

#用处
用来生成图片，甚至可以让图片做加减法，如戴眼镜的男人减去男人加上女人就能生成戴眼镜的女人。也可以根据你随便画的几笔草图来生成风景图

强化学习Deep Q Network

Posted on 2018-08-25 | Edited on 2018-08-26

Deep Q Network能用了训练游戏AI

##神经网络的作用
每一个state，以及在此之下每个action所拥有的Q值，通过输入state而得到action。但如今问题太复杂，而由state、action、Q组成的表格太大了。
所以利用神经网络来代替表格，将状态和动作作为神经网络输入，分析得出Q，然后state对应Q最大的action。

##更新神经网络
如何训练：首先需要a1，a2正确的Q值，namely，现实Q。同样，还需要一个估计Q来实现神经网络的更新，所以神经网络的参数就是老NN参数加学习率alpha乘以现实Q与估计Q的差距。
s2–>NN–>Q(s2,a1)&Q(s2,a2)
上面算出的就是估计Q，选取Q估计中最大值的action换取环境奖励。而现实Q也包括从神经网络分析出来的两个估计Q，不过这两个Q上针对下一步在s‘的估计。
a(现实Q-估计Q)–>NN

##DQN两大利器
Experience replay和Fixed Q-targets使DQN变得无比强大。
Q learning上一种off-policy离线学习法，能学当前经历的，也能学过去经历过的，甚至学习别人的经历。
Fixed Q-targets上一种打乱相关性的机理，使用这个能利用两个结构相同但参数不同的神经网络，预测估计Q
的神经网络具备最新参数，而预测现实Q的神经网络使用参数上很久以前的。
