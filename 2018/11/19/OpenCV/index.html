##OpenCV的安装配置及依赖包出错的解决

如果不是闲着蛋疼h就不要手动了

sudo apt-get install libopencv-dev python-opencv

搞定了，如果是python3就j改成python3-opencv

因为我是手动编译安装所以得自己手动先安装依赖包

sudo apt-get update
sudo apt-get install build-essential cmake git libgtk2.0-dev pkg-config libavcodec-dev libavformat-dev libswscale-dev

然后按照上面指示安装依赖包只有装libcurl3成功，其他都是连环错误。
经大佬指出是我这个ubuntu18.04系统源的问题。

uname -a

得到系统信息Linux ubuntu 4.15.0-38-generic #41-Ubuntu SMP Wed Oct 10 10:59:38 UTC
2018 x86_64 x86_64 x86_64 GNU/Linux 

cat /etc/apt/sources.list

得

# deb cdrom:[Ubuntu 18.04.1 LTS _Bionic Beaver_ - Release amd64 (20180725)]/ bionic main restricted
deb http://mirrors.aliyun.com/ubuntu/ trusty main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ trusty-security main restricted universe multiverse

deb-src http://mirrors.aliyun.com/ubuntu/ trusty-proposed main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ trusty-backports main restricted universe multiverse

源本应该是bionic但却是trusty，说明我这个系统镜像有问题要把trusty切换成bionic，接下来是解决环节

sudo sed -i 's/trusty/bionic/g' /etc/apt/sources.list
sudo apt update

搞定，然后继续配置环境，重回第一步
sudo apt-get install build-essential cmake git libgtk2.0-dev pkg-config libavcodec-dev libavformat-dev libswscale-dev

再

sudo apt-get install python-dev python-numpy libtbb2 libtbb-dev libjpeg-dev libpng-dev libtiff-dev libjasper-dev libdc1394

然后出现errorE:unable to locate libjasper-dev，解决方法如下

sudo add-apt-repository "deb http://security.ubuntu.com/ubuntu xenial-security main"
sudo apt update
sudo apt install libjasper1 libjasper-dev

其中libjasper1是libjasper-dev的依赖包，然后再次

sudo apt-get install python-dev python-numpy libtbb2 libtbb-dev libjpeg-dev libpng-dev libtiff-dev libjasper-dev libdc1394

Done，接下来下载OpenCV的源码

wget -O opencv.zip https://github.com/Itseez/opencv/archive/3.2.0.zip
unzip opencv.zip

#编译安装
进入OpenCV文件夹中创建build目录进行编译
cd openvn-3.2.0
mkdir build
cd build
cmake -D CMAKE_BUILD_TYPE=RELEASE -D CMAKE_INSTALL_PREFIX=/usr.local ..
make
sudo make install

安装成功后具体测试办法看网上
